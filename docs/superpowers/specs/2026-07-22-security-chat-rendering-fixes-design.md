# 设计：安全边界与聊天轨迹修复

日期：2026-07-22
状态：已批准，待实施

## 背景

只读调试确认了六类问题：首次启动接口可绕过正常登录；知识库、文档和 Wiki 接口缺少一致的认证与租户隔离；子 Agent 的 Markdown 被作为纯文本完整展开；工具失败被展示为成功；流恢复超时没有持久化终态；知识库详情重复拉取包含大体积 metadata 的文档列表。

本设计采用契约级定向修复。目标是在不重构无关模块、不覆盖当前工作区已有改动的前提下，统一资源安全边界、完善流式事件契约，并提供适合知识工作台的紧凑可折叠执行轨迹。

## 已批准决策

1. `auto-setup` 只在数据库没有任何用户且显式开启初始化配置时可用；已有用户后必须正常登录。
2. 未认证请求返回 `401`；已认证但访问其他租户资源时返回 `404`，不泄露资源是否存在。
3. 子 Agent 运行中默认展开；进入成功、失败或取消终态后默认折叠；用户可以逐项切换。
4. 采用定向 helper 和明确的数据契约，不引入覆盖全项目的权限中间件重构。

## 安全边界

### 首次初始化

- 新增显式布尔配置 `ALLOW_AUTO_SETUP`，默认关闭。
- `POST /api/v1/auth/auto-setup` 仅在 `ALLOW_AUTO_SETUP=true` 且用户表为空时创建首个管理员并签发令牌。
- 用户表非空、配置关闭或初始化竞争失败时返回 `401`，不为现有用户签发令牌。
- 前端路由守卫保留初始化尝试，但失败后跳转 `/login`；现有用户不能通过页面刷新取得新令牌。

### 租户资源查询

- 在共享后端模块中提供认证上下文和租户限定查询 helper，资源查询必须同时包含资源 ID、当前租户以及软删除条件。
- 知识库、Knowledge、Chunk、KnowledgeTag、WikiPage、WikiFolder 及相关批量操作使用同一边界。
- 集合接口先验证认证，再按当前租户构造 QuerySet。
- 详情、更新、删除、重解析、取消、移动、复制、上传、Wiki 编辑等写路径不得从未限定租户的对象继续执行。
- 认证失败返回 `401`；限定 QuerySet 查不到对象时由 `404` 响应统一处理。

## 聊天轨迹与 Markdown

### 子 Agent 组件

- 新建独立 `ActorTrace.vue`，由 `AssistantMessage.vue` 传入 `actor_traces`。
- 每个 Actor 使用语义化按钮作为摘要行，并提供 `aria-expanded`、稳定的展开图标、名称、状态、耗时。
- `running` 和 `pending` 初始展开；`idle/success/failure/cancelled` 初始折叠。用户手动切换后，在该组件生命周期内保留选择。
- 展开区显示错误或 Markdown 输出；失败状态使用明确错误样式，不使用成功图标。

### Markdown 渲染

- 扩展共享 `renderMarkdownLite()`，支持标题、强调、链接、无序/有序列表、围栏代码块、水平分隔线和 GFM 风格表格。
- 渲染前转义原始 HTML，链接仅允许安全协议，并使用 `rel="noopener noreferrer"`。
- 主回答、Wiki 和子 Agent 继续共享同一渲染器及 `.markdown-lite` 样式，不引入新的 Markdown 依赖。
- 表格在窄屏下允许容器内横向滚动，不扩大页面整体宽度。

## SSE 工具事件契约

- 后端 `tool_call` 和 `tool_result` 事件保留 `tool_call_id`、`name`、`iteration`、`arguments`、`output`、`error`、`duration_ms`。
- 取消 SSE 层无提示的 300 字二次截断；引擎现有展示上限作为唯一边界，并在发生截断时提供明确标记。
- 前端以 `tool_call_id` 为主键匹配结果；仅对不含 ID 的兼容事件回退到“同名且运行中”的调用。
- 工具结果模型保存 `error`，失败项使用 `failed` 状态和错误图标；成功项才使用 `done` 状态。
- 工具详情可折叠，避免检索结果和 Actor 结果同时撑开时间线。

## 流恢复终态

- `continue-stream` 遇到生成错误、恢复超时或已有流丢失且数据库消息仍未完成时，将可展示错误写入 `content/rendered_content`，并持久化 `is_completed=True`。
- 持久化和 SSE 使用相同错误文本；随后发送 `error` 和 `done` 事件。
- 页面刷新后不再重复恢复同一失败消息。
- `chat/views.py` 与历史兼容实现必须保持同一行为，避免路由切换导致契约差异。

## 文档列表性能

- 增加知识文档轻量列表序列化，排除 `metadata`、内部文件路径等详情字段。
- 文档集合响应包含 `status_counts` 和必要的标签计数；计数基于过滤前、当前租户与当前知识库范围聚合。
- `KnowledgeDetail.vue` 只发一次分页请求，并直接使用响应聚合结果，不再调用 `page_size=1000` 的 `loadAllDocs()`。
- 详情、处理和编辑接口继续返回完整数据，不改变这些接口的能力。

## 错误处理与兼容性

- API Key 与 Bearer Token 均遵循相同租户边界。
- 前端保留对不含 `tool_call_id` 的旧 SSE 事件兼容，但新后端测试要求 ID 不得丢失。
- 已保存的 `actor_traces` 不需要数据迁移；组件根据现有 `status/last_outcome/output/error/metadata.duration_ms` 渲染。
- `auto-setup` 配置关闭时，空数据库用户只能通过显式部署初始化流程开启该配置后创建。

## 测试与验收

### 后端

- 无认证读取知识库文档、文档详情和 Wiki 返回 `401`。
- 租户 B 对租户 A 的所有资源读写返回 `404`，租户 A 的同类操作正常。
- `ALLOW_AUTO_SETUP=false` 拒绝初始化；开启且无用户时允许一次；创建用户后再次调用返回 `401`。
- SSE 工具结果包含 ID、错误和耗时，且不发生 300 字二次截断。
- 恢复超时后数据库消息包含错误内容并处于完成状态。
- 文档列表不返回大体积 metadata，并返回状态聚合。

### 前端

- Markdown 表格、分隔线、代码块和安全转义生成预期 HTML。
- 已完成 Actor 默认折叠，运行 Actor 默认展开，按钮正确维护 `aria-expanded`。
- 工具失败显示失败状态和错误文本，并按 `tool_call_id` 更新正确调用。
- 知识库详情只发一次文档列表请求。

### Playwright

- 真实历史会话中的 Actor 输出生成标题和表格 DOM，完成项默认折叠且可展开。
- 模拟工具失败时显示失败图标和错误文本。
- 1440px 桌面和 390px 移动端无页面级横向溢出、控制台错误或失败请求。

## 非目标

- 不重构全部 Django 视图为 DRF。
- 不更换认证令牌模型。
- 不重做聊天页面视觉系统。
- 不修改 Agent 推理策略、知识检索算法或已有知识内容。
- 不提交或回退与本次修复无关的工作区改动。
